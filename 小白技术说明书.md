# 准二维梳齿密封计算程序 v2 小白技术说明书

这份说明书面向第一次接触本程序的人，目标是讲清楚：程序要什么输入、内部按什么步骤计算、输出结果怎么看，以及想改算例时该改哪里。

## 1. 这个程序是做什么的

梳齿密封可以理解成一串很窄的“齿顶间隙”和较宽的“腔室”交替排列。气体从高压入口流向低压出口时，会在每个齿顶被节流，在每个腔室里发生扩散、耗散和压力损失。

本程序把复杂的二维/三维流动简化成工程上可快速计算的准二维网络模型。v2 增加了 CoolProp 真实物性库，可以计算超临界二氧化碳：

```text
入口压力
  -> 第 1 个齿顶节流
  -> 第 1 个腔室损失
  -> 第 2 个齿顶节流
  -> 第 2 个腔室损失
  -> ...
  -> 出口压力
```

它主要回答这些问题：

- 给定入口压力、温度和出口压力，泄漏量是多少？
- 每一级齿后的压力大概是多少？
- 哪个齿附近流速最高？
- 有没有齿接近临界流或已经临界？
- 这个设计有没有明显风险提示？

程序适合做方案初筛和工程估算，不替代 CFD 或实验标定。

## 3. 怎么运行一个算例

打开 PowerShell，进入程序目录：

```powershell
cd "E:\01项目资料\07干气密封\01程序\迷宫密封\v1\准二维计算程序"
```

设置 Python 查找路径：

```powershell
$env:PYTHONPATH='src'
```

运行超临界 CO2 示例算例：

```powershell
python -m q2d_labyrinth.cli examples/sco2_5teeth.json --out outputs/sco2_5teeth
```

运行后，屏幕会显示类似：

```text
Case: sco2_5teeth_real_gas
Mass flow: 0.5090964 kg/s
Outlet pressure target: 8000000.000 Pa
Outlet pressure calculated: 7999952.810 Pa
Max Mach: 0.2284 at tooth 5
Choked teeth: none
Output: ...\outputs\sco2_5teeth
```

这表示程序已经算完，并且把结果写到了 `outputs/sco2_5teeth`。

## 4. 输入文件里都有什么

示例输入文件是：

```text
examples/air_5teeth.json
```

它是一个 JSON 文件，可以用记事本、VS Code 或其他文本编辑器打开。

### 4.1 case

```json
"case": {
  "name": "air_5teeth_fast_design"
}
```

这里是算例名称。它只用于显示和报告，不影响计算。

### 4.2 fluid

```json
"fluid": {
  "type": "ideal_gas",
  "name": "air",
  "gas_constant": 287.05,
  "gamma": 1.4,
  "cp": 1005.0,
  "viscosity": 1.8e-5
}
```

这里定义工质。第一版只实现了理想气体空气。

主要参数含义：

- `gas_constant`：气体常数，空气约为 `287.05 J/(kg K)`
- `gamma`：比热比，空气常用 `1.4`
- `cp`：定压比热，空气约为 `1005 J/(kg K)`
- `viscosity`：动力黏度，空气常温约为 `1.8e-5 Pa s`

### 4.3 geometry

```json
"geometry": {
  "tooth_count": 5,
  "diameter": 0.05,
  "clearance": 0.0002,
  "tooth_width": 0.001,
  "tooth_height": 0.001,
  "cavity_length": 0.002,
  "cavity_height": 0.001
}
```

这里定义密封几何。单位全部是米。

| 参数 | 含义 |
|---|---|
| `tooth_count` | 齿数 |
| `diameter` | 轴径或平均直径 |
| `clearance` | 径向间隙 |
| `tooth_width` | 单个齿的轴向宽度 |
| `tooth_height` | 齿高 |
| `cavity_length` | 两齿之间腔室的轴向长度 |
| `cavity_height` | 腔室径向高度 |

最敏感的参数通常是 `clearance`。间隙变大，泄漏量一般会明显变大。

### 4.4 boundary

```json
"boundary": {
  "inlet_pressure": 1000000.0,
  "inlet_temperature": 300.0,
  "outlet_pressure": 500000.0
}
```

这里定义边界条件。

- `inlet_pressure`：入口压力，单位 Pa
- `inlet_temperature`：入口温度，单位 K
- `outlet_pressure`：目标出口压力，单位 Pa

注意：出口压力必须小于入口压力，否则气体不会按当前方向泄漏。

### 4.5 coefficients

```json
"coefficients": {
  "Cd_tooth": 0.72,
  "K_expansion": 0.5,
  "K_contraction": 0.3,
  "theta_carryover": 0.35,
  "eta_recovery": 0.2
}
```

这些是经验系数，用来把复杂流动压缩成工程模型。

| 参数 | 简单理解 |
|---|---|
| `Cd_tooth` | 齿顶流量系数，越大表示越容易漏 |
| `K_expansion` | 从齿顶喷出进入腔室时的突扩损失 |
| `K_contraction` | 从腔室再进入下一个齿顶时的突缩损失 |
| `theta_carryover` | 腔室内残余动能携带系数 |
| `eta_recovery` | 腔室静压恢复系数 |

如果有 CFD 或实验数据，最应该校准的是这些系数。

### 4.6 solver

```json
"solver": {
  "tolerance_pressure": 1.0e-5,
  "max_iterations": 100
}
```

这里控制迭代精度。

- `tolerance_pressure`：出口压力相对误差容许值
- `max_iterations`：最大迭代次数

一般不用改。

## 5. 程序内部是如何计算的

下面按运行顺序讲。

### 第一步：读取输入文件

命令行入口在：

```text
src/q2d_labyrinth/cli.py
```

当你运行：

```powershell
python -m q2d_labyrinth.cli examples/air_5teeth.json --out outputs/air_5teeth
```

程序会先调用 `case.py` 读取 JSON 输入文件，并把里面的内容整理成几个对象：

- `SealGeometry`：几何参数
- `Boundary`：入口/出口边界
- `Coefficients`：经验系数
- `IdealGas`：空气物性模型
- `SolverOptions`：求解器设置

同时程序会做基础检查，例如：

- 齿数必须大于等于 1
- 压力和温度必须大于 0
- 出口压力必须小于入口压力
- 间隙、齿宽、腔长等几何尺寸必须大于 0

如果输入明显错误，程序会直接报错，不继续算。

### 第二步：先猜一个泄漏量

真实泄漏量一开始不知道。程序采用“猜泄漏量 -> 算出口压力 -> 和目标出口压力比较 -> 再调整泄漏量”的办法。

这就像调水龙头：

- 猜的泄漏量太小，沿程压降不够，算出来的出口压力偏高。
- 猜的泄漏量太大，沿程压降太大，算出来的出口压力偏低。
- 不断二分调整，直到算出来的出口压力接近目标出口压力。

这个过程在 `solver.py` 的 `solve_case()` 中完成。

### 第三步：按齿逐级向下游推进

对于每一个猜测的泄漏量，程序会从入口开始，一个齿一个齿往下游算。

每一级的计算顺序是：

```text
上游状态
  -> 齿顶节流
  -> 判断是否临界
  -> 计算齿顶速度、马赫数、雷诺数
  -> 腔室突扩/突缩损失
  -> 腔室压力恢复
  -> 得到下一级入口状态
```

这部分在 `solver.py` 的 `_march()` 中完成。

### 第四步：齿顶节流怎么理解

齿顶间隙很窄，像一个环形小孔。气体通过它时会加速并降压。

程序使用可压缩孔口流量公式，并乘以齿顶流量系数 `Cd_tooth`。

简化理解：

```text
齿顶越宽松，越容易漏
入口压力越高，越容易漏
出口压力越低，越容易漏
Cd_tooth 越大，越容易漏
```

当压力比足够低时，齿顶可能发生临界流。临界流意味着该齿顶处流速接近声速，继续降低下游压力也不会让该齿顶质量流量无限增大。

输出里的 `is_choked` 就是这个判断结果。

### 第五步：腔室损失怎么理解

气体从齿顶喷入腔室后，会扩散、混合、耗散。程序用几个工程系数描述这件事：

```text
压力损失 = 突扩损失 + 突缩损失 - 压力恢复
```

其中：

- `K_expansion` 越大，腔室突扩损失越大。
- `K_contraction` 越大，进入下一级齿顶前的损失越大。
- `eta_recovery` 越大，腔室静压恢复越强。
- `theta_carryover` 越大，残余动能携带到下一级越多。

这就是“准二维”的主要来源：程序没有直接解二维流场，而是用这些系数表达腔室中的二维流动效果。

### 第六步：为什么要迭代质量流量

用户给的是入口压力和出口压力，不是泄漏量。

但程序按齿推进时，需要先知道质量流量，才能算每一级速度和压降。所以程序必须反过来找一个质量流量，使最后算出来的出口压力等于用户指定的出口压力。

程序使用二分法：

```text
设定泄漏量下限 = 0
设定泄漏量上限 = 第一级齿顶临界流量的 1.2 倍

不断取中间值：
  如果算出来的出口压力偏高，说明泄漏量猜小了
  如果算出来的出口压力偏低，说明泄漏量猜大了

直到出口压力误差满足 tolerance_pressure
```

二分法不花哨，但稳定可靠，适合第一版工程程序。

## 6. 输出结果怎么看

运行后会生成三个主要文件。

### 6.1 summary.json

这是总结果，适合程序读取或后续批处理。

里面包括：

- `mass_flow`：泄漏量，单位 kg/s
- `outlet_pressure_target`：目标出口压力
- `outlet_pressure_calculated`：程序算出来的出口压力
- `iterations`：迭代次数
- `max_mach`：最大马赫数
- `max_mach_tooth`：最大马赫数所在齿
- `choked_teeth`：临界流齿位
- `warnings`：警告信息

### 6.2 teeth.csv

这是逐齿结果，可以用 Excel 打开。

主要列如下：

| 列名 | 含义 |
|---|---|
| `tooth` | 第几个齿 |
| `p_up` | 该齿上游压力 |
| `T_up` | 该齿上游温度 |
| `p_down` | 该齿和腔室之后的下游压力 |
| `T_down` | 下游温度 |
| `dp` | 该级压力降 |
| `dp_fraction` | 该级压降占总压降比例 |
| `mach` | 该齿附近马赫数 |
| `reynolds` | 雷诺数 |
| `is_choked` | 是否临界 |
| `choking_margin` | 距临界的压力比裕度 |

看压力分布时，重点看 `p_up`、`p_down`、`dp`。

看流动风险时，重点看 `mach`、`is_choked`、`choking_margin`。

### 6.3 report.md

这是给人看的简要报告。

里面会列出：

- 泄漏量
- 目标出口压力和计算出口压力
- 最大马赫数
- 临界流齿位
- 警告信息

## 7. 想改一个新算例怎么办

最简单做法是复制示例文件：

```powershell
copy examples/air_5teeth.json examples/my_case.json
```

然后打开 `examples/my_case.json`，修改：

- 齿数：`geometry.tooth_count`
- 直径：`geometry.diameter`
- 间隙：`geometry.clearance`
- 腔长：`geometry.cavity_length`
- 入口压力：`boundary.inlet_pressure`
- 出口压力：`boundary.outlet_pressure`
- 入口温度：`boundary.inlet_temperature`

再运行：

```powershell
python -m q2d_labyrinth.cli examples/my_case.json --out outputs/my_case
```

## 8. 常见改动会带来什么影响

下面是一般趋势，具体结果仍以计算为准。

| 改动 | 常见影响 |
|---|---|
| 间隙 `clearance` 增大 | 泄漏量明显增大 |
| 齿数 `tooth_count` 增大 | 泄漏量降低 |
| 腔长 `cavity_length` 增大 | 通常泄漏量降低，但过大后收益有限 |
| 入口压力增大 | 泄漏量增大 |
| 出口压力降低 | 泄漏量增大，更容易临界 |
| `Cd_tooth` 增大 | 泄漏量增大 |
| 损失系数增大 | 压降增大，泄漏量降低 |

## 9. 看到警告该怎么办

程序不会因为普通警告停止计算，因为工程估算经常会遇到边界工况。但警告意味着结果可信度可能下降。

常见警告包括：

- 几何无量纲参数超出推荐范围
- `Cd_tooth` 不在常见范围
- 马赫数接近 1
- 迭代次数用完但压力误差还没达到要求

处理建议：

1. 先检查输入单位是否正确，尤其是毫米是否误写成米。
2. 检查入口/出口压力是否写反。
3. 对重要工况，用 CFD 或实验数据校准 `Cd_tooth`、`K_expansion`、`K_contraction`、`theta_carryover`。
4. 如果某一级接近临界流，重点关注该齿附近的设计。

## 10. 程序当前版本的局限

第一版有意保持简单清晰，因此有一些明确限制：

- 只支持空气理想气体。
- 不考虑真实气体效应。
- 不考虑两相流。
- 不考虑转子旋转。
- 不考虑偏心。
- 不考虑换热。
- 不直接求解真实二维速度场。
- 腔室二维流动影响由经验系数表示。

这些不是程序错误，而是第一版工程模型的适用边界。

## 11. 一句话总结程序流程

程序先读入几何、边界和系数，然后不断猜泄漏量；每猜一次，就从入口开始按“齿顶节流 + 腔室损失”逐级算到出口；如果算出的出口压力不等于目标出口压力，就继续调整泄漏量，直到两者足够接近，最后输出泄漏量和逐齿状态。
